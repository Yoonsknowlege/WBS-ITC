Reproduction package for:

> Lee, T.S. & Lee, Y. (2026). *Auditable denominator design for patent field delineation: A WBS-ITC codebook-plus-ledger approach*.

- Permanent archive: [10.5281/zenodo.20425593](https://doi.org/10.5281/zenodo.20425593) (Zenodo concept DOI; always resolves to the latest version)

## 1. Package scope

This package supports **exact downstream reconstruction** of the
manuscript's reported numeric outputs from the released adjudicated mapping
layer. It does **not** support deterministic replay of upstream Lens.org
retrieval or consensus adjudication, both of which are documented as
inspectable but not replayable.

Status flags are carried jointly by `phase2_453_families.json` and
`decision_ledger.csv`. The authoritative status flags for accounting are
the `screening_category` and `analysis_set_443` columns of the decision
ledger:

| `screening_category` value | N families | Meaning |
|---|---|---|
| `included_classified` | 373 | ITC-classified subset (≥ 1 adjudicated tag) |
| `domain_external_active` | 70 | Active layer; plausible transfer relevance, no adequate fit within the frozen 15-domain codebook |
| `false_match` | 9 | Excluded from active layer |
| `audit_only` | 1 | Below-threshold record retained only for ledger transparency |

For status accounting, use `screening_category`, not the legacy `status`
column (which collapses the 70 / 9 / 1 distinction into a single
`domain_external` label and is retained only for backward compatibility).

## 3. Python environment

Tested with **Python 3.10** and **Python 3.11**.

Default (broad version ranges):

```bash
pip install -r requirements.txt
```

For strict environment fidelity to the snapshot used in the manuscript,
use the pinned lock file:

```bash
pip install -r requirements-lock.txt
```

Conda users can alternatively reproduce the environment with:

```bash
conda env create -f environment.yml
conda activate wbs-itc
```

`requirements.txt` pins NumPy < 3, Matplotlib < 4, and seaborn < 1;
`requirements-lock.txt` and `environment.yml` pin exact versions
(NumPy 1.26.4, Matplotlib 3.8.4, seaborn 0.13.2, pandas 2.2.2)
verified to reproduce the MANIFEST values on CPython 3.11.

## 4. Execution order

The scripts have no dependencies on each other; they all read directly from
`data/`.

**One-command reconstruction.** A wrapper script is provided that runs
the full reconstruction in the expected order:

```bash
bash run_all.sh
```

This regenerates the body tables, Online Resource 1 quantitative
diagnostics, and figures listed in MANIFEST.md. The Table S15
audit-protocol scripts (`scripts/build_audit_sample.py`,
`scripts/score_audit.py`) are intentionally **not** invoked by
`run_all.sh`; they should be run only when the reviewer-executable blind
audit protocol is being exercised by an external coder.

A typical full reconstruction run (equivalent to `run_all.sh`) is:

```bash
# Main analytical accounting and rule-to-adjudicated diagnostic outputs + numeric cross-checks
python scripts/recompute_tables_from_json.py

# Online Resource 1 quantitative diagnostics
python scripts/concordance_diagnostic.py
python scripts/baseline_comparison.py
python scripts/anchor_sensitivity.py
python scripts/nonseed_bridging.py
python scripts/sensitivity_checks.py
python scripts/seed_only_baseline.py    # Online Resource 1, Table S14
python scripts/expanded_baseline.py     # Online Resource 1, Table S16 (expanded)
python scripts/table_s12_profiles.py    # Online Resource 1, Table S12 (numeric)
python scripts/domain_external_partition.py   # Online Resource 1, Table S7a (coarse)
python scripts/build_long_form_ledger.py      # data/decision_ledger_long.csv

# Online Resource 1, Table S15 — reviewer-executable blind audit protocol (optional;
# no author-internal audit results are reported in the manuscript)
python scripts/build_audit_sample.py    # writes data/audit_sample.csv (blind)
                                        # and data/audit_sample_key.csv (author-only)
# After an external coder fills in data/audit_sample.csv:
python scripts/score_audit.py

# Body Figs. 2-4 and Online Resource 1 Fig. S1
mkdir -p .mplconfig
MPLCONFIGDIR=.mplconfig python scripts/generate_all_figures.py
```

## 6. What is exactly reproduced

The scripts regenerate the main analytical accounting, rule-to-adjudicated diagnostics, Jaccard outputs, and body Figs. 2–4 (the released package figure versions are bundled in `figures/`), and Online
Resource 1 Tables S6a, S8, S9, S10, S13, S14, S16, Fig. S1, and the **numeric
components** of Table S12. The Table S12 numeric components produced by
`scripts/table_s12_profiles.py` are: adjacency pair, Jaccard, intersection
size, dominant CPC top-3 prefixes within the intersection, and 2020-or-later
share. The interpretive **keyword summaries** in Table S12 ("core technology
keywords" column) are documented in the manuscript but **not**
script-generated.

**Table S15 audit protocol.** `scripts/build_audit_sample.py` and
`scripts/score_audit.py` support the Table S15 reviewer-executable blind
audit protocol. The protocol is provided so independent execution by a reviewer or
third party is procedurally possible; **no author-internal audit results
are reported in the manuscript**, because the authors cannot blind-code
their own adjudicated layer. Any agreement score generated by running
these scripts on an external coder's completed sheet should be interpreted
as a bounded reliability check of the released operational layer, not as
external gold-standard validation.

**Fig. 2 structure.** Body Fig. 2 is a two-panel figure with separate
y-axis scales: Panel A reports the 15 ITC-domain adjudicated tag counts
(N = 373 ITC-classified families; 795 tags total; multi-tagging
permitted), and Panel B reports a single grey bar showing the 70 active
domain-external families (N = 70 active families, count not tags). The
two panels are kept on separate axes so that tag counts and family
counts are not visually equated. The 70-family count is cross-checkable
from `decision_ledger.csv` by counting rows with
`screening_category = domain_external_active`. The further partition of
the 70 active domain-external families by extension direction
(foundation engineering, crushing / size reduction, etc.) is reported in
Online Resource 1 Table S7a only and is not shown in body Fig. 2; a
coarse CPC-prefix diagnostic for that partition is provided by
`scripts/domain_external_partition.py` (see the script docstring for
the limits of that diagnostic).

## 7. What is inspectable but not replayed

The package documents but does not deterministically replay:

- **Upstream Lens.org retrieval**: index coverage and saved-search states
  may change over time. The descriptive query provenance and query
  anatomy are in `data/lens_query.txt`; exact replay of the Lens.org
  saved-search state is not claimed.
- **Consensus adjudication**: family-level adjudication actions
  (`add`, `suppress`, `add+suppress`, `keep`, etc.) are inspectable in
  `data/decision_ledger.csv` but the consensus process itself is not
  re-executed.

## 8. What is not claimed

The released mapping layer is the **operational reference layer for this
study only** — not an external gold standard or coder-independent
validation of the field boundary.

The package does not claim:

- A full census of the construction-relevant patent field.
- Engineering convergence or validated subsystem coupling between
  domains (Jaccard adjacencies are read as classification-space patterns).
- Technology readiness, maturity, or national/organizational leadership
  rankings.
- Coder-independent inter-rater reliability validation (κ, PABAK, α are
  reported as procedural diagnostics, not generalisable reliability
  estimates).
- Comparative superiority over alternative delineation methods. The
  Online Resource 1 Table S14 within-layer comparison and the Table S16
  expanded within-layer benchmark are **within-layer** on the same
  443-family active layer, not external benchmarks.

## 9. AI-assisted tools

General-purpose large language models were used as AI-assisted tools,
restricted to language editing, readability support, and non-decisional
review-order prioritization. No AI-generated ranking was used to exclude
records, assign WBS–ITC labels, compute results, or interpret findings.
All scientific decisions, coding, computations, and interpretations are
the authors' responsibility.

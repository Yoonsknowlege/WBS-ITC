## 1. Package scope

This package supports **computational reconstruction** of the manuscript's
data-derived numeric outputs (tables and figures) from the released dataset
and the claim-reviewed WBS-ITC assignment. It does **not** support
deterministic replay of upstream Lens.org retrieval or the first-claim
review, both of which are documented as inspectable but not replayable.

The quantities compare the deterministic rule-hit layer (CPC / title-abstract)
with the claim-reviewed WBS-ITC assignment as **within-corpus
agreement / divergence**, not as classifier-performance metrics.

For the study's methodology, definitions, scope, claims, and limitations, see the associated manuscript and its Online Resource 1.

Status flags are carried jointly by `phase2_453_families.json` and
`assignment_table.csv`. The authoritative status flags for accounting are
the `status_category` and `analysis_set_443` columns of the assignment table:

| `status_category` value | N families | Meaning |
|---|---|---|
| `included_classified` | 373 | ITC-classified subset (>= 1 assigned tag) |
| `domain_external_active` | 70 | Active layer; plausible transfer relevance, no adequate fit within the released 15-domain specification |
| `false_match` | 9 | Excluded from active layer |
| `screening_log_only` | 1 | Below-threshold record retained only for record transparency |

For status accounting, use `status_category`, not the legacy `status`
column (retained only for backward compatibility).

**Data-dictionary note.** Key released columns: `assignment_hit` (the
claim-reviewed WBS-ITC assignment indicator, in `assignment_table_long.csv`),
`assignment_action` (the family-level keep / add / suppress summary; the manuscript refers to the suppress action as remove, in
`assignment_table.csv`), and the `status_category` flags described above.

**Version-numbering note.** The reproduction-package version (this release:
v0.1.3) and the WBS-ITC specification version recorded inside
`data/itc_codebook.json` `_meta.version` are tracked on independent
timelines and do not need to match.

## 3. Python environment

Tested with **Python 3.10** and **Python 3.11**.

Default (broad version ranges):

```bash
pip install -r requirements.txt
```

For strict environment fidelity, use the pinned lock file:

```bash
pip install -r requirements-lock.txt
```

Conda users can alternatively reproduce the environment with:

```bash
conda env create -f environment.yml
conda activate wbs-itc
```

## 4. Execution

The scripts have no dependencies on each other; they all read directly from
`data/`. A wrapper runs the full reconstruction in order:

```bash
bash run_all.sh
```

This regenerates the body tables, Online Resource 1 quantitative tables, and
figures listed in MANIFEST.md.

## 6. What is reproduced

The scripts regenerate the analytical accounting, rule-hit/assignment
agreement, top Jaccard co-tagging pairs, body Figs. 2-5, and Online
Resource 1 Tables S6a, S8, S9, S10, S13, S14, S16, Fig. S1, and the
**numeric components** of Table S12 (adjacency pair, Jaccard, intersection
size, dominant CPC top-3 prefixes, 2020-or-later share). The interpretive
keyword summaries in Table S12 are documented in the manuscript but **not**
script-generated.

**Fig. 2 structure.** Body Fig. 2 is a two-panel figure with separate
y-axis scales: Panel A reports the 15 ITC-domain assignment tag counts
(N = 373 ITC-classified families; 795 tags; multi-tagging permitted), and
Panel B reports a single bar for the 70 active domain-external families
(count, not tags). The further partition of the 70 families by extension
direction is reported in Online Resource 1 Table S7a only.

## 7. What is inspectable but not replayed

- **Upstream Lens.org retrieval**: index coverage and saved-search states
  may change over time. Descriptive query provenance is in
  `data/lens_query.txt`; exact replay of the saved-search state is not claimed.
- **First-claim review**: family-level assignment actions (`add`, `suppress`,
  `keep`, etc.) are inspectable in `data/assignment_table.csv`, but the
  review process itself is not re-executed.
